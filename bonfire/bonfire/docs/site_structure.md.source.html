   1  ## Site Structure
   2  
   3  Bonfire's site is structured with three goals in mind:
   4  
   5  - Keep as much of Bonfire out of the application folder.
   6  - Keep as much of your code (and ours) outside of the web root as possible for added security.
   7  - Help you keep your project organized amid the pressures of everyday development.
   8  
   9  Hopefully, this page will give you an idea of the reasoning and purpose behind the structure.

  62  ### Application
  63  
  64  The __application__ folder is where you will do most of your work. This contains all of your controllers, libraries, helpers, models, and any modules you might use.
  65  
  66  As much as possible, Bonfire stays away from this folder. However, there are a few files that we cannot move out of the application folder due to restrictions in either CodeIgniter or the HMVC solution that we use. In most cases, those files are just fine for you to edit, anyway. These are files like *MY_Controller*, etc.
  67  
  68  Most of the folders with the application folder are the standard CodeIgniter folders you would expect. There are a few new ones that Bonfire adds.
  69  
  70  
  71  #### DB folder
  72  
  73  The db folder stores two things. The first are any backups that you create using the built-in Database Backup funtionality. The second folder, *migrations*, holds the migration files that are specific to your application as a whole.
  74  
  75  #### Modules folder
  76  
  77  This folder is where you should create all of your own modules. It is also where Bonfire creates module-related files for you. Each module should have a folder of it's own unique name here.
  78  
  79  
  80  
  81  
  82  ### Bonfire
  83  
  84  The bonfire folder holds Bonfire, itself, as well as the codeigniter system files.
  85  
  86  - helpers - has additional helper files that Bonfire provides for extra functionality.
  87  - libraries - all of Bonfire's libraries, like the Assets and Template libraries.
  88  - migrations - the migration files that are specific to the core of Bonfire only.
  89  - modules - all of our core modules are stored here, to keep them separate from your modules and make upgrading a bit simpler.
  90  
  91  
  92  
  93  ### Install
  94  
  95  This folder simply holds the files necessary to run the swanky installer that gets Bonfire setup for you. Once installed, feel free to delete this folder.
  96  
  97  ### Public
  98  
  99  This folder is where you should point your domain name to. It holds the files and folders that should be accessible from the web.
 100  
 101  - assets - a place to put all of your scripts, images, and styles that can be used site-wide.
 102  - install - a simple folder that holds the index file that kicks off the installer. Once installed, a new file called *installed.txt* will be written here that will keep the installer off of your back and let you run the site without deleting files. Very handy during development when you know that you'll need the installer later on.
 103  - themes - holds the admin and default front-end themes for now. Also where you should put your own themes.
 104  
 105  ## Reverting To Traditional Structure
 106  
 107  In some situations, you may prefer a more traditional folder structure that has all of the files and folders located in the web root. This might be due to restrictions on shared hosting, or becuase you prefer easier installation for your customers. This can be easily done by moving the files and folders from the public folder into the same folder as the others. This would leave a folder structure like
 108  
 109      application/
 110      assets/
 111      bonfire/
 112      index.php
 113      install/
 114      tests/
 115      themes/
 116  
 117  After you've moved the files, you need to edit the main <tt>index.php</tt> file to let CodeIgniter know where to find the application and system folders.
 118  
 119      $path = '.'
 120  
 121  Changing this one variable will reset all of the other paths.
