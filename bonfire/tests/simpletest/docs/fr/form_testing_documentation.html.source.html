  53          <h2>
  54  <a class="target" name="submit"></a>Valider un formulaire simple</h2>
  55              <p>
  56                  Lorsqu'une page est téléchargée par <span class="new_code">WebTestCase</span>
  57                  en utilisant <span class="new_code">get()</span> ou <span class="new_code">post()</span>
  58                  le contenu de la page est automatiquement analysé.
  59                  De cette analyse découle le fait que toutes les commandes
  60                  à l'intérieur de la balise &lt;form&gt; sont disponibles
  61                  depuis l'intérieur du scénario de test.
  62                  Prenons par exemple cet extrait de code HTML...
  63  <pre>
  64  &lt;form&gt;
  65      &lt;input type="text" name="a" value="A default" /&gt;
  66      &lt;input type="submit" value="Go" /&gt;
  67  &lt;/form&gt;
  68  </pre>
  69                  Il ressemble à...
  70              </p>
  71              <p>
  72                  <form class="demo">
  73                      <input type="text" name="a" value="A default">
  74                      <input type="submit" value="Go">
  75                  </form>
  76              </p>
  77              <p>
  78                  Nous pouvons naviguer vers ce code, via le site
  79                  <a href="http://www.lastcraft.com/form_testing_documentation.php">LastCraft</a>,
  80                  avec le test suivant...
  81  <pre>
  82  class SimpleFormTests extends WebTestCase {<strong>
  83      function testDefaultValue() {
  84          $this-&gt;get('http://www.lastcraft.com/form_testing_documentation.php');
  85          $this-&gt;assertField('a', 'A default');
  86      }</strong>
  87  }
  88  </pre>
  89                  Directement après le chargement de la page toutes les commandes HTML
  90                  sont initiées avec leur valeur par défaut, comme elles apparaîtraient
  91                  dans un navigateur web. L'assertion teste qu'un objet HTML
  92                  avec le nom "a" existe dans la page
  93                  et qu'il contient la valeur "A default".
  94              </p>
  95              <p>
  96                  Nous pourrions retourner le formulaire tout de suite...
  97  <pre>
  98  class SimpleFormTests extends WebTestCase {
  99      function testDefaultValue() {
 100          $this-&gt;get('http://www.lastcraft.com/form_testing_documentation.php');
 101          $this-&gt;assertField('a', <strong>new PatternExpectation('/default/')</strong>);
 102      }
 103  }
 104  </pre>
 105                  Mais d'abord nous allons changer la valeur du champ texte.
 106                  Ce n'est qu'après que nous le transmettrons...
 107  <pre>
 108  class SimpleFormTests extends WebTestCase {
 109  
 110      function testDefaultValue() {
 111          $this-&gt;get('http://www.my-site.com/');
 112          $this-&gt;assertField('a', 'A default');<strong>
 113          $this-&gt;setField('a', 'New value');
 114          $this-&gt;clickSubmit('Go');</strong>
 115      }
 116  }
 117  </pre>
 118                  Parce que nous n'avons spécifié ni attribut "method"
 119                  sur la balise form, ni attribut "action",
 120                  le scénario de test suivra le comportement classique d'un navigateur :
 121                  transmission des données avec une requête <em>GET</em>
 122                  vers la même page. En règle générale SimpleTest essaie d'émuler
 123                  le comportement typique d'un navigateur autant que possible,
 124                  plutôt que d'essayer d'attraper des attributs manquants sur les balises.
 125                  La raison est simple : la cible d'un framework de test est
 126                  la logique d'une application PHP, pas les erreurs
 127                  -- de syntaxe ou autres -- du code HTML.
 128                  Pour les erreurs HTML, d'autres outils tel
 129                  <a href="http://www.w3.org/People/Raggett/tidy/">HTMLTidy</a>
 130                  devraient être employés, ou même n'importe lequel des validateurs HTML et CSS
 131                  déjà sur le marché.
 132              </p>
 133              <p>
 134                  Si un champ manque dans n'importe quel formulaire ou si
 135                  une option est indisponible alors <span class="new_code">WebTestCase::setField()</span>
 136                  renverra <span class="new_code">false</span>. Par exemple, supposons que
 137                  nous souhaitons vérifier qu'une option "Superuser"
 138                  n'est pas présente dans ce formulaire...
 139  <pre>
 140  <strong>Select type of user to add:</strong>
 141  &lt;select name="type"&gt;
 142      &lt;option&gt;Subscriber&lt;/option&gt;
 143      &lt;option&gt;Author&lt;/option&gt;
 144      &lt;option&gt;Administrator&lt;/option&gt;
 145  &lt;/select&gt;
 146  </pre>
 147                  Qui ressemble à...
 148              </p>
 149              <p>
 150                  <form class="demo">
 151                      <strong>Select type of user to add:</strong>
 152                      <select name="type">
 153                          <option>Subscriber</option>
 154                          <option>Author</option>
 155                          <option>Administrator</option>
 156                      </select>
 157                  </form>
 158              </p>
 159              <p>
 160                  Le test suivant le confirmera...
 161  <pre>
 162  class SimpleFormTests extends WebTestCase {
 163      ...
 164      function testNoSuperuserChoiceAvailable() {<strong>
 165          $this-&gt;get('http://www.lastcraft.com/form_testing_documentation.php');
 166          $this-&gt;assertFalse($this-&gt;setField('type', 'Superuser'));</strong>
 167      }
 168  }
 169  </pre>
 170                  La sélection ne sera pas changée si la nouvelle valeur n'est pas une des options.
 171              </p>
 172              <p>
 173                  Voici la liste complète des objets supportés à aujourd'hui...
 174                  <ul>
 175                      <li>Champs texte, y compris les champs masqués (hidden) ou cryptés (password).</li>
 176                      <li>Boutons submit, en incluant aussi la balise button, mais pas encore les boutons reset</li>
 177                      <li>Aires texte (textarea) avec leur gestion des retours à la ligne (wrap).</li> 
 178                      <li>Cases à cocher, y compris les cases à cocher multiples dans un même formulaire.</li>
 179                      <li>Listes à menu déroulant, y compris celles à sélections multiples.</li>
 180                      <li>Boutons radio.</li>
 181                      <li>Images.</li>
 182                  </ul>
 183              </p>
 184              <p>
 185                  Le navigateur proposé par SimpleTest émule les actions
 186                  qui peuvent être réalisées par un utilisateur sur
 187                  une page HTML standard. Javascript n'est pas supporté et
 188                  il y a peu de chance pour qu'il le soit prochainement.
 189              </p>
 190              <p>
 191                  Une attention particulière doit être porté aux techniques Javascript
 192                  qui changent la valeur d'un champ caché : elles ne peuvent pas être
 193                  réalisées avec les commandes classiques de SimpleTest.
 194                  Une méthode alternative est proposée plus loin.
 195              </p>

 197          <h2>
 198  <a class="target" name="multiple"></a>Champs à valeurs multiples</h2>
 199              <p>
 200                  SimpleTest peut gérer deux types de commandes à valeur multiple :
 201                  les menus déroulants à sélection multiple et les cases à cocher
 202                  avec le même nom à l'intérieur même d'un formulaire.
 203                  La nature de ceux-ci implique que leur initialisation
 204                  et leur test sont légèrement différents.
 205                  Voici un exemple avec des cases à cocher...
 206  <pre>
 207  &lt;form class="demo"&gt;
 208      <strong>Create privileges allowed:</strong>
 209      &lt;input type="checkbox" name="crud" value="c" checked&gt;&lt;br&gt;
 210      <strong>Retrieve privileges allowed:</strong>
 211      &lt;input type="checkbox" name="crud" value="r" checked&gt;&lt;br&gt;
 212      <strong>Update privileges allowed:</strong>
 213      &lt;input type="checkbox" name="crud" value="u" checked&gt;&lt;br&gt;
 214      <strong>Destroy privileges allowed:</strong>
 215      &lt;input type="checkbox" name="crud" value="d" checked&gt;&lt;br&gt;
 216      &lt;input type="submit" value="Enable Privileges"&gt;
 217  &lt;/form&gt;
 218  </pre>
 219                  Qui se traduit par...
 220              </p>
 221              <p>
 222                  <form class="demo">
 223                      <strong>Create privileges allowed:</strong>
 224                      <input type="checkbox" name="crud" value="c" checked><br>
 225                      <strong>Retrieve privileges allowed:</strong>
 226                      <input type="checkbox" name="crud" value="r" checked><br>
 227                      <strong>Update privileges allowed:</strong>
 228                      <input type="checkbox" name="crud" value="u" checked><br>
 229                      <strong>Destroy privileges allowed:</strong>
 230                      <input type="checkbox" name="crud" value="d" checked><br>
 231                      <input type="submit" value="Enable Privileges">
 232                  </form>
 233              </p>
 234              <p>
 235                  Si nous souhaitons désactiver tous les privilèges sauf
 236                  ceux de téléchargement (Retrieve) et transmettre cette information,
 237                  nous pouvons y arriver par...
 238  <pre>
 239  class SimpleFormTests extends WebTestCase {
 240      ...<strong>
 241      function testDisableNastyPrivileges() {
 242          $this-&gt;get('http://www.lastcraft.com/form_testing_documentation.php');
 243          $this-&gt;assertField('crud', array('c', 'r', 'u', 'd'));
 244          $this-&gt;setField('crud', array('r'));
 245          $this-&gt;clickSubmit('Enable Privileges');
 246      }</strong>
 247  }
 248  </pre>
 249                  Plutôt que d'initier le champ à une valeur unique,
 250                  nous lui donnons une liste de valeurs.
 251                  Nous faisons la même chose pour tester les valeurs attendues.
 252                  Nous pouvons écrire d'autres bouts de code de test
 253                  pour confirmer cet effet, peut-être en nous connectant
 254                  comme utilisateur et en essayant d'effectuer une mise à jour.
 255              </p>

 257          <h2>
 258  <a class="target" name="hidden-field"></a>Formulaires utilisant Javascript pour changer un champ caché</h2>
 259              <p>
 260                  Si vous souhaitez tester un formulaire dépendant de Javascript
 261                  pour la modification d'un champ caché, vous ne pouvez pas
 262                  simplement utiliser setField().
 263                  Le code suivant <em>ne fonctionnera pas</em> :
 264  <pre>
 265  class SimpleFormTests extends WebTestCase {
 266      function testEmulateMyJavascriptForm() {
 267          <strong>// This does *not* work</strong>
 268          $this-&gt;setField('a_hidden_field', '123');
 269          $this-&gt;clickSubmit('OK');
 270      }
 271  }
 272  </pre>
 273                  A la place, vous aurez besoin d'ajouter le paramètre supplémentaire
 274                  du formulaire à la méthode clickSubmit() :
 275  <pre>
 276  class SimpleFormTests extends WebTestCase {
 277      function testMyJavascriptForm() {
 278          <strong>$this-&gt;clickSubmit('OK', array('a_hidden_field'=&gt;'123'));</strong>
 279      }
 280  
 281  }
 282  </pre>
 283              </p>
 284              <p>
 285                  N'oubliez pas que de la sorte, vous êtes effectivement en train
 286                  de court-circuitez une partie de votre application (le code Javascript
 287                  dans le formulaire) et que peut-être serait-il plus prudent
 288                  d'utiliser un outil comme
 289                  <a href="http://selenium.openqa.org/">Selenium</a> pour mettre sur pied
 290                  un test complet.
 291              </p>
 292          
 293          <h2>
 294  <a class="target" name="brut"></a>Envoi brut</h2>
 295              <p>
 296                  Si vous souhaitez tester un gestionnaire de formulaire
 297                  mais que vous ne l'avez pas écrit ou que vous n'y avez
 298                  pas encore accès, vous pouvez créer un envoi de formulaire à la main.
 299  <pre>
 300  class SimpleFormTests extends WebTestCase {
 301      ...<strong>    
 302      function testAttemptedHack() {
 303          $this-&gt;post(
 304                  'http://www.my-site.com/add_user.php',
 305                  array('type' =&gt; 'superuser'));
 306          $this-&gt;assertNoUnwantedPattern('/user created/i');
 307      }</strong>
 308  }
 309  </pre>
 310                  En ajoutant des données à la méthode <span class="new_code">WebTestCase::post()</span>,
 311                  nous émulons la transmission d'un formulaire.
 312                  D'ordinaire, vous ne ferez cela que pour parer au plus pressé,
 313                  ou alors si vous espérez un tiers (javascript ?) transmette le formulaire.
 314                  Il reste quand même exception : si vous souhaitez vous protégez
 315                  d'attaques de type "spoofing".  
 316              </p>
