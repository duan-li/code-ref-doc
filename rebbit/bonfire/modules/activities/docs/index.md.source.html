   1  ## Monitoring Site Activity
   2  
   3  The Activities module provides a simple way to record activity from any module.
   4  
   5  ### log_activity() helper function
   6  
   7  To log activities you can use the <tt>log_activity()</tt> helper function that will always be available. This helper function checks the value of the config item <tt>enable_activity_logging</tt> and, if enabled, will autoload the activity_model and log the activity. This allows you to quickly disable logging in your application without needing to rewrite portions of your code.
   8  
   9  The first parameter is the ID of the user the activity is being recorded about. The second parameter is the activity message that you would like logged. The third parameter is the name of the module. Any spaces should be converted to and underscore.
  10  
  11      $status = 'just turned the lights out.';
  12  
  13      log_activity( $user_id, $status, 'my_module');
  14  
  15  ### Initializing the Model
  16  
  17  Like most other models in CodeIgniter, the Activities Model is initialized in your controller by using the <tt>$this->load->model</tt> function:
  18  
  19      $this->load->model('activities/activity_model');
  20  
  21  Once loaded, the Activities features will be avialable by using <tt>$this->activity_model</tt>.
  22  
  23  ### log_activity()
  24  
  25  To record an activity for later viewing, use the <tt>log_activity</tt> method. The first parameter is the ID of the user the activity is being recorded about. The second parameter is the activity message that you would like logged. The third parameter is the name of the module. Any spaces should be converted to and underscore.
  26  
  27      $status = 'just turned the lights out.';
  28  
  29      $this->activity_model->log_activity( $user_id, $status, 'my_module');
  30  
  31  ## Viewing Activities
  32  
  33  Once logged, activities can be viewed in the Admin area and filtered by user or module.
